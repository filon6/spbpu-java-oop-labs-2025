# ООП лабораторные

Репозиторий с лабораторными по ООП.  
Год выполнения: **2025**

## Условия лабораторных

## Лабораторная 1: Принципы ООП и паттерн стратегия

``В компьютерной игре герой (класс Hero) может перемещаться между двумя точками (метод move) различными способами: идти пешком, ехать на лошади, лететь и т. п. Реализовать классы, позволяющие выбирать и менять в ходе выполнения программы способ перемещения героя, используя паттерн “стратегия” (strategy). Продемонстрировать работу реализованных классов.``

## Лабораторная 2: Annotation и Reflection 	

``Написать аннотацию с целочисленным параметром. 
Создать класс, содержащий публичные, защищенные и приватные методы (2-3 каждого вида) с параметрами, аннотировать любые из них.
Вызвать из другого класса все аннотированные защищенные и приватные методы столько раз, сколько указано в параметре аннотации. 
Вызывающий методы код не должен зависеть от количества и типов параметров этих методов.``

## Лабораторная работа 4: String, StringBuilder, Exceptions

Требуется реализовать консольную программу-переводчик со следующими требованиями:

1. При запуске программы выполняется чтение словаря из файла.  
   Формат словаря: слово или выражение | перевод
2. После загрузки словаря считывается входной текстовый файл, для которого выполняется перевод.
3. Перевод осуществляется по следующим правилам:
    - регистр букв не учитывается;
    - если слово отсутствует в словаре, оно выводится без перевода;
    - если существует несколько подходящих вариантов перевода, выбирается вариант с **максимальной длиной левой части**  
  (например, при наличии переводов для `look` и `look forward to` должен быть выбран второй вариант).
4. Результат перевода выводится в консоль.
5. Необходимо создать и использовать пользовательские исключения:
    - `InvalidFileFormatException` — неверный формат словаря;
    - `FileReadException` — ошибки чтения файла (файл не существует, отсутствует доступ и т.д.).

## Лабораторная работа 5: Stream API

С использованием **только Stream API** реализовать следующие методы:

- метод, приводящий все строки в списке в верхний регистр и добавляющий к ним префикс «_new_»;
- метод, возвращающий список квадратов всех встречающихся только один раз элементов списка;
- метод, принимающий на вход коллекцию строк и возвращает все строки, начинающиеся с заданной буквы, отсортированные по алфавиту;
- метод, принимающий на вход коллекцию и возвращающий ее последний элемент или кидающий исключение, если коллекция пуста;
- метод, принимающий на вход массив целых чисел, возвращающий сумму чётных чисел или 0, если чётных чисел нет;
- метод, преобразовывающий все строки в списке в Map, где первый символ – ключ, оставшиеся – значение;
